手机登录 + 云端 Todo（给朋友用）

目标：账户隔离（不同手机号看到自己的清单）+ 云端存储（更安全/可跨设备）+ 仍然保持前端体验接近现在。

交互形态（前端）

入口 A：进入即要求登录

未登录只显示登录面板。登录后进入 Todo。最简单、数据隔离最清晰。

入口 B：可浏览 / 但编辑需要登录

未登录可看示例 Todo（只读），点击编辑/新增时弹登录。更“作品集式”展示。

登录流程（手机号 + 验证码）
输入手机号
→
发送验证码
→
输入验证码
→
登录成功（拿到 userId）

说明：发送短信这一步必须依赖第三方（Firebase/Supabase/Twilio 等），否则无法“真验证码”。我们会封装成统一接口，保证 UI 不变、未来可替换供应商。

云端方案（后端）

方案 1（推荐）：Supabase

  • Auth：支持手机号 OTP（一般需要配置短信通道，如 Twilio）
  • 数据：Postgres + Row Level Security（按 userId 隔离）
  • 前端：直接用 supabase-js，几乎不写后端代码
  • 部署：前端仍用 GitHub Pages；云端在 Supabase

方案 2：Firebase

  • Auth：手机号登录成熟（短信成本/风控也要配置）
  • 数据：Firestore（按 uid 隔离）
  • 优点：上手快；缺点：数据模型/查询方式不同、迁移成本略高
数据流（建议）
GitHub Pages（React）
↔
Auth（手机号 OTP）
↔
DB（按 userId 隔离 Todo）

本地 localStorage 可以保留作为离线缓存，但最终以云端为准（登录后自动同步）。
